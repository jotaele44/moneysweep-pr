# Data Policy — moneysweep-pr

**Version:** 1.0 · 2026-05-15

---

## 1. Data Classification

| Class | Examples | Committed to Git? | Notes |
|-------|---------|-------------------|-------|
| **Public bulk data** | USASpending CSV exports, Grants.gov XML | No | Too large; downloaded on demand |
| **Public API data** | FEC filings, SAM.gov entities, FDIC banks | No | API keys may be required |
| **Semi-public portal exports** | HUD DRGR reports, FEMA PA worksheets | No | Require operator login; stored in approved dropzones only |
| **Enrichment outputs** | SAM parent resolution, entity master | No | May contain vendor PII; gitignored |
| **Test fixtures** | `tests/fixtures/r5/*.csv` | Yes | Minimal synthetic samples only |
| **Registry files** | `registries/source_registry.json` | Yes | Metadata only, no personal data |
| **Reports** | `reports/*.md`, `reports/*.csv` | Yes | Derived summaries; no raw PII |
| **API keys / secrets** | SAM_API_KEY, LDA_API_KEY | **Never** | `.env` is gitignored; use `.env.example` as template |

---

## 2. What Is Never Committed

The following are **permanently gitignored** and must never be committed:

- `.env` (real API keys)
- `data/staging/processed/**` (processed source data)
- `data/staging/raw/**` (raw downloaded data)
- `data/normalized/` (normalized parquet outputs)
- `data/staging/processed/enrichment/` (may contain vendor PII)
- `data/raw/*.csv` (raw source CSVs)
- PDF/document corpora (`data/raw/documents/**/*.pdf`)
- Logs (`data/logs/*.log`)

Violations are caught by `scripts/scan_for_secrets.py` (run in pre-commit or CI).

---

## 3. Secret Handling

All API keys must follow the rules in `docs/SECRET_HANDLING_POLICY.md`:

- Keys live only in `.env` (gitignored) or in a separate `moneysweep-pr-Secrets/` directory outside the repository.
- No key, token, password, or credential may appear in any committed file.
- The `.env.example` template is the only committed reference to key names.
- If a key is accidentally committed, it must be **rotated immediately** — git history rewriting does not remove exposure risk from forks/clones.

Audit command:
```bash
python3 scripts/scan_for_secrets.py
```

## 5. Data Retention and Cleanup

- Downloaded raw data files are ephemeral. They should be re-downloadable from source at any time.
- Processed outputs in `data/staging/processed/` are intermediate artifacts and may be regenerated by re-running the pipeline.
- The `data/manual_import_dropzone/` paths contain operator-delivered files that may not be re-downloadable. **Do not delete these without operator confirmation.**
- Test fixtures in `tests/fixtures/` are permanent and version-controlled.

---

## 6. PII and Sensitive Data

This pipeline processes government contracting data. The data may incidentally contain:
- Vendor employee names (from SAM.gov registrations)
- Individual lobbyist registrations (from LDA, Cabilderos)
- Beneficiary counts (from CHIP, Medicaid, WIC) — no individual-level data

No individual-level health, financial, or identifying records are ingested or stored.  
The enrichment output in `data/staging/processed/enrichment/` should be treated as potentially sensitive and is gitignored.

---

## 7. Repository Access

The repository is currently public. This is intentional — all data sources are public domain or publicly licensed, and no secrets or PII are committed.

Before making the repository private or transferring to an organization, review:
- Whether any derived data files were accidentally committed (`git log --diff-filter=A -- 'data/**'`).
- Whether all secrets are confirmed absent (`python3 scripts/scan_for_secrets.py`).
